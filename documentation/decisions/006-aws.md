# ADR-006 — AWS track (2026-07-10)

## 006.1 Terraform for IaC
**Context:** the tool choice is itself practice; Terraform is the most
job-demanded skill; CDK/Pulumi fit owner's TS skills but transfer less.
**Decision:** Terraform (state in S3 + DynamoDB lock, plan/apply discipline,
community VPC/EKS modules). **Consequences:** industry-default practice;
everything transfers to OpenTofu if licensing ever matters.

## 006.2 EKS as the compute platform
**Context:** EKS maximizes continuity with the v2 kind lab (same kustomize
overlays, same drills, real nodes/AZs); ECS is cheaper but forks the mental
model. **Decision:** EKS; ECS explicitly not pursued (a scoped ECS exercise
could be a future opportunistic item). **Consequences:** k8s practice
compounds; control-plane cost handled by session-scoped lifecycle (006.4).

## 006.3 Managed swaps: S3 and RDS; the rest self-hosted
**Context:** per-component fidelity/cost/practice trade. **Decision:**
**S3 replaces MinIO** (same API the code speaks; presigned URLs/IAM/lifecycle
practice at ~zero cost) and **RDS replaces in-cluster Postgres** (parameter
groups, backups, failover drills — the most job-relevant swap).
Redis, RabbitMQ, Mongo stay self-hosted in-cluster. **Consequences:** two
managed-service surfaces in Terraform; broker internals stay drill-accessible.

## 006.4 Cost guardrails: budget alarm + session lifecycle + TTL reaper
**Context:** PRD demands idle ≈ $0 and known session cost. **Decision:** all
three: AWS Budgets with alerts (50/80/100%), `make aws-up`/`aws-down` as the
only workflow (nothing persists but tfstate and ECR), and TTL tags with a
scheduled reaper killing stragglers. Spot instances: not adopted now —
optional later. **Consequences:** three independent layers against surprise
bills; forgetting a teardown is annoying, not expensive.

## 006.5 Dedicated AWS account, pinned region
**Context:** reaper/destroy tooling in a personal account is a foot-gun.
**Decision:** a lab-only account (Organizations or standalone), one pinned
region, least-privilege credentials for the tooling. **Consequences:**
destroy-everything tools become safe; cost tracking is trivially scoped.

## 006.6 Route53 + ACM on a lab domain
**Context:** drills need reachable endpoints; tunnels are unrealistic and
fiddly. **Decision:** cheap lab domain, Terraform-managed Route53 + ACM +
ALB ingress; endpoints exist only during sessions. **Consequences:** real
DNS/TLS/ingress practice; ~$12/yr + pennies.

## 006.7 Deploys: make first, GitHub Actions OIDC within v5
**Context:** laptop-driven terraform/kubectl suffices to start; the OIDC→ECR
→EKS pipeline is itself high-value practice. **Decision:** first sessions run
via make; before v5 exits, the Actions OIDC pipeline ships (no long-lived
keys), make remains the fallback. **Consequences:** fast start AND the
professional pipeline, sequenced inside one phase.
